# Personal Reflection

This project strengthened my ability to connect physical theory, numerical methods, and software implementation into a single workflow that can be tested and improved iteratively.

Key learning outcomes:

- **Modeling judgment matters:** Not every physically detailed feature is worth its computational or data-fitting cost.
- **Validation is essential:** Benchmarking against experimental concentration data was critical for deciding which approximations were acceptable.
- **Iteration improves clarity:** Several explored features were intentionally removed after testing, leading to a clearer and more robust model.
- **Practical realism is a balance:** The final framework is intentionally less detailed than full direct simulations, but more feasible for repeated studies and parameter sweeps.

Future directions I would pursue:

- stronger uncertainty quantification for tuned parameters
- broader validation datasets across brew methods
- improved fines transport representations with adaptive local topology updates
- tighter integration between simulation outputs and brew-recommendation tools
